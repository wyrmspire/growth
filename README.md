# growth

Growth project workspace.

## Quick map

- `index.html` — front-end entry point
- `server.ts` — local server/backend entry point
- `package.json` — scripts and dependencies
- `MVP_SCOPE.md` — current scope boundary
- `PRODUCT_DESIGN.md` — product/design notes
- `SYSTEM_ARCHITECTURE.md` — technical architecture notes
- `PROJECT_RULES.md` — local project constraints/conventions
- `board.md` / `lanes.md` / `future.md` — planning and backlog notes

## Notes

- Treat this folder as an active local project workspace.
- Prefer updating the scoped docs above instead of creating overlapping notes when possible.
- Keep local-only secrets/config out of committed docs; use example files like `.env.local.example` for shape only.
